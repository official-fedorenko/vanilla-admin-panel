# Личная Админ-Панель и Блог (Vanilla Node.js & SQLite)

Этот проект представляет собой облегченную, производительную и красивую систему управления контентом (CMS) и личный кабинет, построенные на **чистом Node.js (без фреймворков)** и базе данных **SQLite3**. Фронтенд написан на чистом HTML, CSS и Vanilla JS с современным дизайном (темная тема, эффекты стекла, градиенты).

---

## 📂 Структура проекта и назначение папок

*   `server.js` — Основной HTTP-сервер на чистом Node.js. Отвечает за маршрутизацию API-запросов, раздачу статических файлов, парсинг кук, авторизацию сессий и загрузку файлов.
*   `db.js` — Конфигурация базы данных SQLite. При первом запуске автоматически создает таблицы и генерирует демонстрационные данные, настройки и 3 дефолтных аккаунта.
*   `db.sqlite` *(игнорируется Git)* — Локальный файл базы данных SQLite3.
*   `uploads/` *(игнорируется Git)* — Директория на диске, куда загружаются медиафайлы через админку.
*   `package.json` — Файл зависимостей (единственная внешняя зависимость — драйвер `sqlite3`).
*   `client/` — **Пользовательская часть сайта (Блог и Личный кабинет)**:
    *   `index.html` — Главная страница сайта (лендинг блога).
    *   `style.css` — Стили пользовательской части сайта.
    *   `app.js` — Логика главного сайта (динамическая подгрузка настроек, статей, отрисовка меню авторизации).
    *   `login.html` / `register.html` — Страницы входа и регистрации пользователей.
    *   `cabinet.html` — **Личный кабинет пользователя**:
        *   Форма редактирования профиля (Email, смена пароля).
        *   **Интерактивный чат поддержки**:
            *   *Обычный пользователь* видит диалог с техподдержкой.
            *   *Администратор / Суперадмин* видит список обращений от всех пользователей (тикеты) и может общаться с каждым пользователем напрямую (интерфейс встроен прямо в панель чата в режиме "inline").
*   `public/` — **Панель управления (Админка для Admin / Superadmin)**:
    *   Доступна по адресу `/admin/` (автоматический редирект на `/admin/login.html`, если сессия не авторизована).
    *   `index.html` — Одностраничное приложение (SPA) панели управления.
    *   `app.js` — Логика админки (управление статьями через Quill Editor, файловый менеджер медиафайлов, редактор системных настроек, просмотр и очистка логов активности, управление пользователями).
    *   `style.css` — Дизайн админки.

## 🛠️ Архитектура и технологии

### 1. Авторизация и Сессии
*   Сессии хранятся в оперативной памяти сервера в виде `Map()` (`token -> user_info`).
*   При входе в систему сервер генерирует криптографический токен (`crypto.randomBytes(32)`), записывает его в сессии и отправляет клиенту в заголовке `Set-Cookie` с флагами `HttpOnly; Path=/; SameSite=Strict`.
*   При каждом запросе к защищенным API сервер считывает куки через `parseCookies()` и проверяет наличие активного токена.
*   *Важно:* При перезапуске сервера сессии сбрасываются (потребуется повторный вход).

### 2. База данных (Схема `db.sqlite`)
*   `users` — пользователи (ID, логин, хэш пароля `pbkdf2Sync`, роль `Superadmin`/`Admin`/`User`, дата регистрации).
*   `articles` — статьи блога (заголовки, текст, статус `published`/`draft`, дата создания).
*   `settings` — динамические настройки сайта по принципу "ключ-значение" (название сайта, контакты, включение регистрации и т.д.).
*   `media` — загруженные медиафайлы (оригинальное имя, сгенерированный путь на диске, размер, MIME-тип).
*   `logs` — лог действий (кто и когда совершил действие в админке или кабинете).
*   `support_messages` — переписка в чате поддержки (ID сообщения, ID тикета `ticket_id`, текст, роль отправителя, статус прочтения `is_read`).

### 3. API Эндпоинты

#### Публичные:
*   `POST /api/auth/login` — Вход в систему.
*   `POST /api/auth/register` — Регистрация нового пользователя (если разрешено в настройках).
*   `GET /api/auth/check-username` — Быстрая проверка занятости логина.
*   `GET /api/public/articles` — Список опубликованных статей для главной страницы.
*   `GET /api/public/settings` — Публичные системные настройки.

#### Личный кабинет (Требуют авторизации):
*   `GET /api/cabinet/me` — Получение данных текущего авторизованного пользователя.
*   `PUT /api/cabinet/profile` — Обновление Email и пароля пользователя.
*   `POST /api/auth/logout` — Выход (удаление куки и сессии).
*   **Чат техподдержки**:
    *   `POST /api/support/send` — Отправка сообщения (для пользователей и гостей).
    *   `GET /api/support/messages` — Получение истории переписки по конкретному тикету.
    *   `POST /api/support/read` — Отметка сообщений в тикете как прочитанных.

#### Панель Администратора (Доступ только для Admin и Superadmin):
*   `GET /api/dashboard/stats` — Общая статистика для главного экрана админки (пользователи, статьи, файлы).
*   `GET /api/support/tickets` — Получение списка всех обращений (тикетов) в поддержку с количеством непрочитанных сообщений.
*   `POST /api/support/reply` — Ответ администратора в конкретный тикет пользователя.
*   `GET/POST/PUT/DELETE /api/crud/articles` — Универсальный CRUD-эндпоинт для управления статьями.
*   `GET/POST/DELETE /api/media` — Загрузка файлов (парсинг `multipart/form-data`), просмотр библиотеки и удаление.
*   `GET/POST /api/settings` — Чтение и перезапись системных настроек.
*   `GET /api/logs` — Просмотр логов активности (Superadmin также может вызвать `DELETE /api/logs` для их очистки).
*   `GET/POST/PUT/DELETE /api/users` — Управление аккаунтами пользователей (доступно только для `Superadmin`).

## 🚀 Как запустить проект

1.  Убедитесь, что на компьютере установлен **Node.js** (версии 16+).
2.  Склонируйте или перенесите папку с проектом.
3.  Установите зависимость `sqlite3` из корня проекта:
    ```bash
    npm install
    ```
4.  Запустите сервер:
    *   В обычном режиме:
        ```bash
        npm start
        ```
    *   В режиме разработки (с автоперезапуском при изменении файлов):
        ```bash
        npm run dev
        ```
5.  Откройте браузер по адресу: **`http://localhost:3080`**

## 🤖 Подсказка для AI (Для новых нейросетей-ассистентов)
> При редактировании файлов фронтенда в папке `client` помните, что скрипты используют Vanilla JS и выполняют запросы через `fetch`. Всегда оборачивайте инициализацию и привязку событий к DOM-элементам в `document.addEventListener('DOMContentLoaded', ...)` во избежание ошибок `TypeError` (связанных с порядком парсинга HTML и скрипта).
> При доработке API в `server.js` помните, что все новые эндпоинты должны терминировать соединение через `res.end()` или хелпер `sendJson(res, status, data)`. Любое незакрытое ветвление в коде сервера приведет к зависанию HTTP-запроса в браузере клиента.
