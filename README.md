# Vanilla Admin Panel + CMS (Node.js + SQLite)

> 🚀 **Beta** — более зрелая версия после альфы. Основной функционал стабилен, но всё ещё рекомендуется внимательно относиться к безопасности и делать бэкапы перед использованием в production.

Лёгкая **self-hosted** админ-панель + CMS + личный кабинет, написанная **на чистом Node.js без фреймворков** и **SQLite**.

Фронтенд — чистый HTML + CSS + Vanilla JS с современным тёмным дизайном (glassmorphism, градиенты, плавные анимации).

**Особенности проекта:**
- Ноль тяжёлых зависимостей (только `sqlite3`)
- Сессии переживают перезапуск сервера
- Хорошая защита регистрации от ботов "из коробки"
- Подходит как основа для личного проекта или небольшого продукта

---

## ✨ Возможности

- **Публичный сайт** (блог)
  - Динамическая загрузка статей и настроек
  - Адаптивный дизайн

- **Личный кабинет** (`/cabinet.html`)
  - Редактирование профиля (email + смена пароля)
  - Загрузка аватарки
  - Полноценный чат с техподдержкой

- **Админ-панель** (`/admin/`)
  - CRUD статей (Quill WYSIWYG редактор + предпросмотр)
  - Медиатека с drag & drop загрузкой и копированием URL
  - Управление пользователями (только Superadmin)
  - Система тикетов поддержки (просмотр всех обращений + ответы)
  - Редактор системных настроек
  - Просмотр и очистка логов действий

- **Безопасность и защита от ботов**
  - PBKDF2 хэширование паролей (120 000 итераций + обратная совместимость)
  - HttpOnly + SameSite куки
  - **Усиленная защита регистрации**:
    - Случайная математическая капча (сложение/вычитание)
    - Honeypot-поля
    - Минимальное время нахождения на странице (~4.5 сек)
    - Жёсткий rate limiting (3 регистрации с одного IP)
    - Серверная верификация всех проверок
  - Rate limiting на логин (защита от брутфорса)
  - Защита от спама в `/check-username`

- **Технические особенности**
  - Сессии переживают перезапуск сервера (хранятся в БД)
  - Автоматический сид дефолтных аккаунтов и настроек
  - Поддержка ролей: `Superadmin` / `Admin` / `User`
  - Загрузка файлов через JSON + base64 (просто и без зависимостей)

## 🚀 Быстрый старт

```bash
# 1. Клонирование и установка
git clone <your-repo>
cd vanilla-admin-panel
npm install

# 2. (Опционально) Скопируйте и настройте переменные окружения
cp .env.example .env
# Отредактируйте .env при необходимости (например, PORT=4000)

# 3. Запуск
npm start                 # production-режим
npm run dev               # с автоперезапуском (рекомендуется для разработки)
```

Откройте в браузере: **http://localhost:3080** (или порт из `.env`)

### Особенности запуска на Windows (PowerShell)

Если `npm run dev` не работает:

- Откройте обычную **Командную строку (cmd)** вместо PowerShell, или
- Выполните:
  ```powershell
  Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
  ```
- Или запускайте напрямую:
  ```powershell
  node --watch server.js
  ```

## 🔐 Безопасность (Beta)

**Важно для бета-версии:**

- **Сразу меняйте все дефолтные пароли** после первого запуска. Логин `superadmin` имеет полный доступ.
- На логин возвращается `securityWarning`, если вы вошли под дефолтным аккаунтом.
- Рекомендуется сразу после входа зайти в раздел **Пользователи** (доступно только Superadmin) и сменить пароли.
- Рассмотрите отключение самостоятельной регистрации (`allow_registration` в Настройках).

Технические меры:
- Пароли хэшируются с помощью `pbkdf2Sync` (120 000 итераций)
- Сессии привязаны к криптографическим токенам (HttpOnly + SameSite=Strict)
- Все чувствительные эндпоинты требуют авторизации + проверки роли
- **Защита регистрации** (математическая капча + honeypot + rate limiting + серверная верификация)
- Rate limiting на критичные эндпоинты (login/register)
- Файлы загружаются с базовой санитизацией имён и ограничением размера (8MB, max 6 файлов)

**Предупреждение:** Проект не предназначен для прямого публичного экспоза без reverse proxy + HTTPS + дополнительного аудита.

## 🛠️ Разработка

- Используется **нативный `node --watch`** (Node 18.11+ / 20+)
- Нет сборки, нет бандлеров
- Весь фронтенд — Vanilla JS + `fetch`
- Тесты: `npm test` (использует встроенный `node:test`)

**Полезные советы для AI-ассистентов:**
- Все инициализации и обработчики событий во фронтенде должны быть внутри `document.addEventListener('DOMContentLoaded', ...)`.
- Любой API-обработчик (в `server.js` или `src/routes/`) **обязан** завершать ответ через `res.end()` или `sendJson(res, status, data)`.
- Новые эндпоинты лучше добавлять до блока раздачи статики.

## ❓ Troubleshooting

- **Порт 3080 занят** — измени константу `PORT` в `server.js`.
- **npm не работает в PowerShell** — см. раздел "Быстрый старт".
- **Не работает загрузка файлов** — убедись, что папка `uploads/` существует (создаётся автоматически).
- **Сессии сбрасываются** — это нормально при `npm run dev` с перезапуском. В продакшене используй `npm start`.
- **Хочу поменять дефолтный адрес/название** — удали `db.sqlite` или измени значения через админку → Настройки.

## ⚠️ Ограничения (Beta)

- Нет пагинации на больших объёмах данных (логи, медиа, пользователи грузятся целиком).
- Rate limiting хранится в памяти (сбрасывается при перезапуске сервера).
- Backend использует лёгкое разделение (`server.js` + `src/routes/*`). Фронтенд админки (`public/app.js`) остаётся одним большим файлом.
- SQLite — не для высоконагруженных проектов.
- Нет встроенной HTTPS (рекомендуется ставить за reverse proxy + HTTPS).
- Минимальное количество тестов (базовые smoke-тесты добавлены в 0.3.0-beta).
- UI и API могут меняться между минорными версиями.

**Beta означает:** проект более зрелый, чем альфа, но всё ещё требует осторожности при использовании в production. Обязательно делайте бэкапы `db.sqlite` и `uploads/`.

### Подготовка к показу / демо

При первом запуске сервер выводит в консоль напоминание про дефолтные пароли.

**Чтобы показать чистую версию людям:**
1. Остановите сервер.
2. `rm db.sqlite` (или удалите файл).
3. `npm start` — создастся свежая БД с тремя дефолтными аккаунтами и тестовыми статьями/настройками.
4. Сразу после входа смените хотя бы пароль `superadmin`.

В текущей альфе уже исправлены несколько заметных багов (раздача `/uploads/`, редактирование статей в админке, редиректы после потери сессии и т.д.).

## 🛠️ Production / Деплой

Проект можно использовать в продакшене для небольших сайтов и личных проектов, но учитывайте:

- Используйте `npm start` (без `--watch`).
- Поставьте reverse proxy (Nginx/Caddy) + HTTPS.
- Делайте регулярные бэкапы `db.sqlite` и папки `uploads/`.
- Для постоянной работы на сервере используйте PM2, systemd или Docker.
- Смените все дефолтные пароли и рассмотрите отключение регистрации (`allow_registration` в настройках).

Пример запуска через PM2:
```bash
pm2 start server.js --name admin-panel
```
